# Structured Canopy

这套视觉语言把复杂系统看成一片被精密编排的“农业能力树冠”：外部驱动力像气候与土壤，底层数据与技术像根系与水脉，核心产品模块像逐层展开的枝叶，最终在业务输出层结出可以被看见的价值。画面不依赖花哨装饰，而是通过层级、方向、密度和留白，让观看者在几秒内看懂“从资源输入到业务结果”的完整路径。整张图必须显得经过无数轮推敲，结构严谨、关系清楚、节奏克制，像由真正有体系感的高级信息设计师 painstakingly 打磨而成。

空间上强调“自上而下、由内向外”的生长逻辑。每一层都要有稳定的视觉重心，彼此之间用清晰但不过度喧哗的连接关系建立秩序。框架图不是把很多盒子堆在一起，而是让每个区域都像被精心校准后的系统部件，间距、边界、箭头、标题和注释都带着 master-level execution 的克制。信息越复杂，越要通过秩序感来减少阅读负担；观者应当一眼看到主干，再逐步进入模块细节。

色彩上采用“农险科技”而非“互联网通用蓝紫”的语汇。深林绿承担主结构与信任感，稻谷金用于重点强化和阶段优先级，雾灰与浅米白提供呼吸感和专业底色，局部以冷青色连接技术与数据属性。颜色必须是经过 painstaking attention 调校后的结果：不艳俗、不飘浮、不廉价，像一套可以直接拿去做售前汇报主视觉的系统语言。对比要明确，但边界要柔和，让专业感先于装饰感。

文字不是主角，而是被纳入结构中的说明性构件。标题需要有明确识别度，正文则服务于扫读与定位，避免形成大段阅读负担。整个成品应看起来像花了大量时间反复修边、对齐、压缩和提纯后留下的最优表达，每一个卡片、标签、分区和连接线都体现出 expert craftsmanship。它不是草图，不是白板图，而是一张可以直接进入方案、汇报或墙面打印的高完成度框架图。
