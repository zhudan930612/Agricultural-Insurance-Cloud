# 农业风险减量产品规划与推进清单

## 2. 收费模式细化

### 2.1 当前收费结构判断
当前判断采用"平台基础费 + 交易型服务费 + 增值服务费"的组合收费模式，其中以核保/核查类服务费作为首阶段主要收入来源，平台建设费和专项报告费作为辅助收入来源。

### 2.2 参考收费拆分

| 收费项 | 适用场景 | 参考定价逻辑 | 当前判断 |
|---|---|---|---|
| 平台建设与接入费 | 新客户首次接入、定制部署、地图与权限体系初始化 | 可按项目一次性收取，建议 20-50 万/省级项目起；如为轻量试点可单独商务打包 | 价格待商务确认 |
| 精准核保服务费 | 承保前/承保中核保核验 | 按服务等级和面积计费：卫星核保单价最低，无人机次之，人工+无人机最高；也可按任务单计费 | 首阶段主要收入来源 |
| 风险减量服务费 | 预警、病虫害监测、过程展示监测 | 建议按季/按农时打包收费，或按承保面积年度订阅收费 | 可作为续费收入来源 |
| 理赔辅助服务费 | 灾后定损、核查、证据报告输出 | 建议按灾次、任务量或受灾面积计费，重大灾害时可采用专项项目制 | 高毛利专项收入 |
| 数据分析与定制报告费 | 招投标支持、经营分析、专项评估报告 | 按报告或专题项目收费 | 辅助收入来源 |

### 2.3 参考定价示例
- 卫星遥感核保：适合作为大范围初筛服务，建议采用低单价、大面积模式，价格区间待商务确认。
- 无人机核保：适合作为重点区域复核服务，建议按架次、区域或任务包计费，价格区间待商务确认。
- 人工+无人机核保：适合作为高争议地块或重点项目核验服务，建议按任务单或高单价面积计费，价格区间待商务确认。

### 2.4 成交案例状态
- 太保新疆项目：当前为重点推进机会，尚不能表述为已成交案例。
- 其他保险公司项目案例：待补充，如未形成正式合同或验收闭环，应统一标注为"待商务确认/待案例沉淀"。

## 3. 技术成熟度与验证

### 3.1 技术成熟度评估

| 技术能力 | 当前成熟度 | 当前判断依据 | 主要风险/限制 | 是否需要专项验证 |
|---|---|---|---|---|
| 卫星遥感识别耕地/作物大类 | P1 可用 | 行业内已有较多遥感识别应用，适合作为大范围初筛和底图识别 | 受云层、时相、作物生长阶段影响，精度受区域差异影响明显 | 需要 |
| 卫星遥感直接识别具体作物类型 | P2 谨慎评估 | 理论可行，但对样本、区域模型和时序影像依赖高 | 跨省泛化能力弱，误识别会直接影响核保可信度 | 强烈需要 |
| 基于遥感估算种植面积 | P1 可用 | 适合做面积估算、异常筛查和趋势对比 | 难替代争议地块的最终人工核验 | 需要 |
| 病虫害智能识别 | P2 谨慎评估 | 适合作为预警线索，不宜直接作为定责定损依据 | 病虫害类型复杂，需结合地面巡查验证 | 强烈需要 |
| 无人机精细核查 | P1 可用 | 技术成熟，适合作为重点区域核查手段 | 调度成本、空域审批、天气影响较大 | 需要 |
| 地面信息员现场核验 | P1 可用 | 最接近真实业务落地方式，可补足遥感盲区 | 人员组织、培训、真实性管控是关键 | 需要 |

### 3.2 技术验证结论
- 现阶段更适合将"卫星遥感识别作物类型""病虫害智能识别"表述为辅助判断能力或规划能力，不宜直接承诺为高精度、全区域稳定可用能力。
- 一期产品应优先承诺"遥感初筛 + 无人机复核 + 人工核验"的组合交付模式。
- 后续需补充试点验证指标，包括识别准确率、召回率、误报率、覆盖区域、验证样本量和人工复核差异率。

### 3.3 建议验证任务
- 选取 1-2 个目标试点区域，完成作物识别和面积估算小样本验证
- 形成卫星识别结果与人工核验结果对比样本
- 验证病虫害识别能否达到“预警线索”级别可用
- 明确无人机调度时效、飞行组织要求和回传周期

## 4. 待确认项分级与推进

### 4.1 待确认项分级表

| 优先级 | 待确认项 | 责任建议 | 建议截止时间 | 影响范围 |
|---|---|---|---|---|
| P0 | 太保合作协议的签署进度和具体合作范围 | 商务负责人 | 招投标方案输出前确认 | 直接影响首单落地和方案边界 |
| P0 | 与国家减灾中心合作状态、协议主体、独家性、数据授权边界 | 商务负责人 + 法务 | 对外方案定稿前确认 | 直接影响核心壁垒表述和合规风险 |
| P0 | 国家减灾中心卫星遥感数据的获取权限、使用成本和交付方式 | 商务负责人 + 技术负责人 | 技术方案定稿前确认 | 直接影响成本模型和交付可行性 |
| P0 | 无人机平台的调度方式、响应时效和费用机制 | 技术负责人 + 交付负责人 | 试点方案确认前 | 直接影响服务承诺和交付时效 |
| P0 | 太保新疆项目的必选功能范围与验收口径 | 产品负责人 + 商务负责人 | 需求范围冻结前 | 直接影响一期范围 |
| P1 | 灾害信息员能否纳入协保任务，以及管理权限归属 | 产品负责人 + 商务负责人 | 一期方案评审前 | 影响地面执行组织模式 |
| P1 | 与保险公司核心系统对接的技术方案 | 技术负责人 | 技术设计前 | 影响导入效率和系统边界 |
| P1 | 遥感识别作物类型、面积测算、病虫害识别的技术验证结果 | 技术负责人 + 产品负责人 | 技术验证结束后 | 影响智能化承诺边界 |
| P1 | 新疆以外的省份试点计划和时间节点 | 商务负责人 | 首单立项后 1 个月内 | 影响复制路径和资源准备 |
| P2 | 产品正式名称是否采用"农保云" | 产品负责人 | 对外品牌材料输出前 | 影响品牌一致性 |
| P2 | 是否需要开发农户端/投保人端 | 产品负责人 | 二期规划前 | 影响产品边界和研发投入 |
| P2 | 勾地块功能的业务优先级和落地范围 | 产品负责人 | 一期复盘后 | 影响中长期数据资产路线 |

### 4.2 分级说明
- P0：不确认就无法稳妥对外承诺或无法冻结一期方案
- P1：影响交付质量和中期方案可信度
- P2：可在一期之后继续判断

### 4.3 周度推进建议
1. 商务与法务优先处理 P0 中的合作边界和授权问题。
2. 产品与技术并行处理 P0/P1 中的一期范围和技术验证问题。
3. 每周更新一次责任人、状态、风险和下一动作。

## 5. 一期推进建议

### 5.1 一期优先范围
- 保单地图化
- 分级核保流程闭环
- 基础风险区划展示
- 理赔核查任务与结果回传

### 5.2 非一期必做
- 农户端
- 勾地块精细功能全面铺开
- 病虫害高精度智能识别承诺
- 全国化复制方案细化
